# srvdog Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a lightweight SSH-tunnel-only server resource dashboard with low-frequency background history, realtime on-demand sampling, export, and clear-history support.

**Architecture:** A single Go service binds to `127.0.0.1:8090`, collects host metrics from `/proc` and filesystem stats, stores coarse history in daily JSONL files, keeps short realtime history in memory, and serves a minimal HTML/JS dashboard plus JSON APIs. The frontend drives high-frequency sampling by sending heartbeats while open.

**Tech Stack:** Go standard library, plain HTML/CSS/JS, JSONL storage, Linux `/proc`, Docker CLI for container summary

```go
package history

type Store struct {
	Dir string
}

func (s Store) CleanupExpired(cutoff time.Time) error {
	entries, err := os.ReadDir(s.Dir)
	if err != nil {
		return err
	}
	for _, entry := range entries {
		if entry.IsDir() {
			continue
		}
		name := entry.Name()
		if len(name) != len("2006-01-02.jsonl") || !strings.HasSuffix(name, ".jsonl") {
			continue
		}
		day, err := time.Parse("2006-01-02", strings.TrimSuffix(name, ".jsonl"))
		if err != nil {
			continue
		}
		if day.Before(cutoff) {
			if err := os.Remove(filepath.Join(s.Dir, name)); err != nil {
				return err
			}
		}
	}
	return nil
}
```
